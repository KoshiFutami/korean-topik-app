# review

マージ前に正しさ・セキュリティ・保守性・テストを確認する。

## 手順

1. `git diff main...HEAD` または `gh pr diff` で差分を確認する。
2. **正しさ**: 要求を満たすか、境界条件・エラーハンドリングは十分か。
3. **セキュリティ**: 認証・認可・入力バリデーション・SQL インジェクション・秘密情報の混入。
4. **保守性**: Controller にビジネスロジックが漏れていないか、レイヤー違反・重複がないか。
5. **テスト**: 変更に対し Feature テストが付いているか。`make test` で確認。
6. **Pint**: `make lint-backend` でコードスタイルを確認。
