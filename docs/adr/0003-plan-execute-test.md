# ADR 0003: Plan → Execute → Test ワークフロー

## ステータス

採用済み

## コンテキスト

AI エージェントが複数ファイルにまたがる変更を行う際、無計画な実装はコードの一貫性を損なうリスクがあります。

## 決定

複数ファイルにまたがる変更・新機能追加の際は以下のフローを必須とします：

1. **Plan**: 変更計画をユーザーに説明し、承認を得る
2. **Execute**: 承認された計画に基づいて実装する
3. **Test**: `make test` でテストが通ることを確認する

## 影響

- 1 ファイルのみのバグ修正は Plan フェーズをスキップ可能
- テストが通らない実装は完了とみなさない
- `make lint-backend` もテスト前に実行すること
